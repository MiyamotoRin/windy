# ゲーム企画書（ドラフト）

## 1. 仮タイトル

**Wind Drift Survival（仮）**
― 風を読み、風を操り、生存する ―

---

## 2. ジャンル

環境操作型アクション / サバイバルアクション
（ワンボタン操作・物理挙動ベース）

---

## 3. コンセプト

本作は、「風」という不可視の環境要因を主軸に据え、
プレイヤーがその流れを読み・操作しながら生存時間を競うアクションゲームである。

操作は極めて単純でありながら、

* 風向き
* 風速
* 障害物の移動
* プレイヤーの慣性

といった複合的要素の相互作用により、
高い判断性と身体感覚的な操作精度を要求する体験を設計する。

---

## 4. ゲーム体験の核（Core Fun）

### 4.1 精密さ × スピード

本作の快楽構造は以下の二軸で成立する。

| 要素   | 内容          |
| ---- | ----------- |
| 精密さ  | 風の強さ・方向の微調整 |
| スピード | 瞬間的な判断と回避反応 |

これは弓道・アーチェリー的な「集中と制御」と、
陸上競技的な「瞬発と反応」を融合した設計思想である。

---

## 5. 操作システム

### ワンボタン操作

* ボタン押下 → 風を発生させる
* 押している時間 → 風力が増加

#### 物理挙動

* 短押し → 弱い推進力
* 長押し → 強い推進力（飛ばされやすい）

### リスク設計

* 強く吹きすぎる → 障害物衝突
* 風向き不一致 → 想定外の移動

---

## 6. 風システム設計

### 6.1 環境風（外乱）

ステージには常時、環境風が存在する。

例：

* 北 → 南へ強風
* 渦状の乱流
* 時間変化する風向

### 6.2 プレイヤー風（操作）

プレイヤーは任意方向へ風を発生可能。

---

## 7. 合成ベクトル挙動

移動方向は以下で決定される。

```
最終移動 = 環境風 + プレイヤー風
```

### 例

| 状況            | 結果      |
| ------------- | ------- |
| 環境風：北→南（強）    | ↓       |
| プレイヤー風：西→東（弱） | ↘       |
| 合成結果          | 右下へ流される |

→ 「意図と結果のズレ」がゲーム性を生む。

---

## 8. ゲームルール

### 勝利条件

* なし（エンドレス）

### 目的

* 生存時間最大化

### 敗北条件

* 障害物衝突
* ステージ外流出

---

## 9. 障害物設計

### 動的障害物

* 風に流される物体
* 回転体
* ランダム落下物

### 相互作用

* 障害物も風の影響を受ける
* 風で押し戻すことも可能

---

## 10. 時間変化システム

環境は時間とともに変動。

#### 例

* 風速増加
* 風向ランダム化
* 突風イベント
* 渦発生

→ プレイ時間が長いほど難易度上昇。

---

## 11. ビジュアル演出案

### 風の可視化

* 流線エフェクト
* 粒子表現
* 光の歪み

### 世界観拡張案

* 幻想空間
* 浮遊遺跡
* 雲海ステージ
* 変形ターゲット

※スポーツ的リアリズムに限定しない。

---

## 12. プレイ感覚デザイン

目指す身体感覚：

* 手ブレ制御感
* 慣性制御感
* 風圧感覚
* 浮遊感

→ 「狙う」ではなく「漂流を制御する」感覚。

---

## 13. 差別化ポイント

| 一般的アクション | 本作      |
| -------- | ------- |
| キャラ操作中心  | 環境操作中心  |
| 移動＝入力    | 移動＝力の合成 |
| 直接制御     | 間接制御    |

---

## 14. 拡張可能性

### システム拡張

* 風属性（熱風・冷風）
* 重量変化
* 空気密度

### モード追加

* 的当てモード（精密射撃）
* レースモード（風走）
* PvP（風押し合い）

---

## 15. 技術実装メモ（初期）

* 2D物理 or 疑似ベクトル演算
* Force 加算モデル
* 慣性減衰係数調整
* 風場ノイズ生成

---

# 総括

本企画は、

* ワンボタン操作という極小入力
* 風という不可視環境
* 力の合成という計算的挙動

を統合し、

**「シンプル操作 × 高判断密度」**
のアクション体験を成立させることを目的とする。
